# Auto SMM Telegram Bot

AI-система для автоматической генерации, публикации и анализа контента в Telegram.

## 🚀 Особенности

✅ **Полностью inline-интерфейс** - все действия через кнопки, без спама сообщений  
✅ **AI-генерация контента** через OpenRouter API (підтримка всех популярних моделей)  
✅ **Система аппрува контента** - модерация постов администратором  
✅ **Автогенерация по расписанию** - фоновый агент для автоматического создания и публикации  
✅ **Аналитика и адаптивное расписание** - AI анализирует эффективность и выбирает лучшее время  
✅ **Многоуровневая тарифная система**  
✅ **Анализ комментариев и автоответы**  
✅ **A/B тестирование контента**  

## 🛠️ Архитектура

```
app/
├── bot/
│   ├── handlers/          # Обработчики команд и callback
│   ├── keyboards/         # Inline клавиатуры
│   ├── middlewares/       # Мидлвари (rate limit, auth)
│   └── states/            # FSM и хранилище состояний
├── core/                  # Конфигурация, база данных, логирование
├── models/                # SQLAlchemy модели
├── ai/                    # Интеграция с OpenRouter
│   ├── openrouter_client.py
│   ├── content_generator.py
│   └── comment_analyzer.py
├── services/              # Бизнес-логика
├── analytics/             # Система аналитики
└── tasks/                 # Celery фоновые задачи
```

## 🔧 Установка и запуск

1. **Скопируйте и заполните переменные окружения:**
```bash
cp .env.example .env
```

2. **Установите зависимости:**
```bash
pip install -r requirements.txt
```

3. **Запустите бота:**
```bash
python main.py
```

## 📋 Система аппрува

1. Пользователь создаёт пост и отправляет "на аппрув"
2. Администратор получает уведомление с кнопками:
   - ✅ Одобрить
   - ❌ Отклонить
   - ✏️ Отредактировать
3. После одобрения пост публикуется или попадает в расписание

## 🤖 Автогенерация

Агент работает в фоновом режиме и:
- Генерирует посты по настроенному расписанию
- Учитывает аналитику для оптимизации времени публикации
- Может сразу публиковать или отправлять на аппрув
- Поддерживает разную частоту и стили контента

## 🔄 Система состояний

Все шаги пользователя сохраняются в Redis. Структура callback_data:
```
action:type:id:param
```
